## 1. 小学组智慧生活要求审计

规则材料中，智慧生活视频强调“用户场景 + 交互体验”。小学组任务主要围绕生活中的基础感知、自动控制、语音触发和节能响应展开。

| 小学组任务 | 规则重点 | 本作品落点 |
| --- | --- | --- |
| 数据采集 | 围绕生活场景采集基础环境或状态数据；采得到、读得稳、表达清楚。 | 采集光照、温湿度、声音、人体经过、烟雾/漏水/火焰状态，并在 OLED 与网页看板同步显示。 |
| 智能控制 | 根据光照、温度或声音阈值自动触发生活设备，不用人为按键假装自动。 | 光线暗自动开灯，温度或声音超过阈值时发出舒适提醒，安全异常触发蜂鸣器、OLED 与网页告警。 |
| 语音交互 | 语音指令精准触发对应功能，并能回读当前状态。 | 网页语音或小智语音意图触发“学习模式”“离家模式”“节能模式”“查询安全状态”等白名单命令。 |
| 节能响应 | 在合适时机关掉或切换到更节能状态，要能说明为什么节能。 | 无人或光照充足时关闭学习灯；只有“有人且光线暗”才允许亮灯，网页显示节能分和节能原因。 |

设计注意事项：

- 每个功能都必须有“传感器变化 -> 本地判断 -> 执行器动作 -> OLED/网页显示”的闭环。
- 小学组可以展示 MQTT 和远程看板，但讲解时要回到生活问题：更安全、更舒适、更节能。
- 语音指令数量控制在 5-7 条，现场稳定比命令多更重要。
- 规则多次提示“演示仅作参考”，所以本方案不照搬视频动作，而是抓住小学组能力本质。

## 2. N16R8 套装要求审计

### 2.1 硬件基线

本作品以 `N16R8 ESP32-S3 主板 + S3 传感器扩展板` 为准。`小学/zhihuishenghuo/` 中的 UNO/Keyes 方案可借鉴场景和讲解，但最终接线、代码、看板都应按 N16R8 执行。

N16R8 S3 扩展板使用白色 `G-V-S` 插槽：

| 标识 | 含义 | 常见线色 |
| --- | --- | --- |
| `G` | GND | 黑色 |
| `V` | 低压电源 | 红色 |
| `S` | GPIO 信号 | 黄/白色 |

基本安全要求：

- 所有模块必须共地。
- ESP32-S3 的 ADC/GPIO 信号不能超过 `3.3V`。
- 继电器只接低压演示负载，不接 `220V`。
- MQ-2 若用 `5V` 加热供电，`AO` 必须先分压/限压到 `0~3.3V` 再进 ADC。
- 上电默认继电器关、蜂鸣器静音；安全告警功能默认保持启用。

### 2.2 推荐 GPIO 合同

以下以 `小学/N16R8_SmartHome` 与 `N16R8_SmartLife_ABCS` 中较一致的 N16R8 口径为主。后续写固件时以此表冻结，避免一份文档一个引脚。

| 类型 | 模块 | 接口/GPIO | 作品角色 | 优先级 |
| --- | --- | --- | --- | --- |
| 感知 | 光敏传感器 | `ADC1 / GPIO1` | 光照采集、自动开灯、节能判断 | 必选 |
| 感知 | MQ-2 烟雾/燃气 AO | `ADC2 / GPIO2` | 厨房空气安全、阈值告警 | 必选 |
| 输入 | 8 键 AD 或阈值旋钮 | 候选 `GPIO3` 或 `GPIO10` | 模式切换、阈值调整 | 建议，需实板标定 |
| 感知 | 声音传感器 | `ADC4 / GPIO4` | 学习/居家噪声提醒 | 建议 |
| 感知 | PIR 人体红外 | `D5 / GPIO5` | 有人/无人、离家安防、节能判断 | 必选 |
| 感知 | 火焰传感器 | `D6 / GPIO6` | 火源异常提醒，不用明火演示 | 必选 |
| 感知 | 水滴/漏水传感器 | `D8 / GPIO8` | 厨卫漏水提醒 | 必选 |
| 执行 | 蜂鸣器 | `D13 / GPIO13` | 短提醒、异常报警 | 必选 |
| 感知 | DHT11/DHT22 | `D14 / GPIO14` | 温湿度采集、舒适阈值提醒 | 必选 |
| 输入 | 触碰/RFID 模拟 | `D21 / GPIO21` | 授权进入或确认键 | 可选 |
| 显示 | OLED | `SDA=GPIO41, SCL=GPIO42` | 本地数据、模式、告警显示 | 必选 |
| 执行 | 继电器/照明 | `GPIO48` | 学习灯或低压灯带 | 必选 |

### 2.3 8 键 AD 的特别审计

当前小学资料里 8 键 AD 存在两个口径：

- `N16R8_SmartHome/HARDWARE_GUIDE.md` 将 8 键 AD 写为 `ADC3 / GPIO3`。
- `N16R8/04.豪华阶段/03.8键AD键盘测试/03_8key.py` 的课程例程使用 `ADC(Pin(10))`，并给出了上、下、左、右、A、B、C、D 的 ADC 区间。

因此本方案不把 8 键 AD 作为第一版必备风险点。建议后续实板先运行 8 键 AD 单模块例程，确认当前扩展板插槽到底对应 `GPIO3` 还是 `GPIO10`，再冻结到固件和网页。若现场时间紧，模式切换可先由网页按钮和语音完成。

## 3. 推荐作品定位

作品名称：**N16R8 智慧安心节能小屋**

一句话定位：

> 这是一个面向小学生家庭生活场景的智能小屋。N16R8 会采集光照、温湿度、声音、人体与安全状态，自动控制低压学习灯和蜂鸣器，并通过 OLED、MQTT 与网页看板把真实数据可视化，让评委同时看到实体小屋和云端状态同步变化。

作品主线：

- 小学规则主线：数据采集 -> 智能控制 -> 语音交互 -> 节能响应。
- 真实硬件主线：N16R8 ESP32-S3 -> 传感器 -> 本地判断 -> 执行器。
- 展示增强主线：Web Serial -> WSS Relay -> MQTT -> Dashboard 可视化。

不建议第一版做成“万能智能家居”。核心讲四件事：

1. 看得见环境：光照、温湿度、声音、有人/无人。
2. 控得动设备：低压学习灯和蜂鸣器，并让 OLED、网页同步反馈。
3. 听得懂指令：少量稳定语音指令映射到安全命令。
4. 说得清节能：为什么开灯、为什么关灯、为什么进入节能状态。

## 4. 功能设计

### 4.1 四个场景模式

| 模式 | 触发方式 | 自动逻辑 | 执行反馈 |
| --- | --- | --- | --- |
| 居家模式 `home` | 默认、网页、语音“回到居家” | 采集环境；有人且光线暗时开灯，安全异常仍优先报警 | 学习灯、OLED、网页状态 |
| 学习模式 `study` | 网页、语音“开始学习”、8 键 A | 光线暗开学习灯；温度或声音超阈值时短提醒 | 学习灯、蜂鸣器、OLED、网页原因提示 |
| 离家模式 `away` | 网页、语音“我要出门”、8 键 C | 关闭学习灯；PIR 检测到人触发入侵提醒 | 蜂鸣器短报警、OLED、网页告警 |
| 节能模式 `energy` | 网页、语音“开启节能”、8 键 D | 仅在有人且光线暗时允许开灯；无人或光照充足立即关灯 | 学习灯、节能分、节能原因、OLED 与网页 |

安全异常优先级高于普通模式：MQ-2 超阈值、漏水、火焰触发时，即使当前是节能模式，也应进入安全提醒状态。

### 4.2 小学任务映射

| 小学任务 | 现场演示动作 | 评委看到的结果 |
| --- | --- | --- |
| 数据采集 | 遮挡光敏、热水杯靠近 DHT、拍手、PIR 前挥手 | OLED 与网页数值同步变化，日志出现真实 telemetry |
| 智能控制 | 学习模式下遮光、升温或制造安全音量内的声音变化 | 灯自动亮；温度或声音超阈值时蜂鸣器短提醒，网页写明触发原因 |
| 语音交互 | 说“开始学习”“我要出门”“家里安全吗”“开启节能” | 模式切换、状态回读、语音记录同步到看板 |
| 节能响应 | 开启节能模式，让光照充足或保持无人 | 学习灯关闭，节能分上升，页面显示节能原因 |

### 4.3 安心安全模块

MQ-2、水滴、火焰和离家 PIR 正式纳入第一版，用真实安全采集与告警闭环支撑“数据采集”和“智能控制”任务：

| 场景 | 传感器 | 判断 | 执行器 |
| --- | --- | --- | --- |
| 厨房烟雾/燃气 | MQ-2 AO `GPIO2` | 超过阈值 | 蜂鸣器短报警，OLED 与网页显示风险来源 |
| 卫生间/厨房漏水 | 水滴 `GPIO8` | 触发 | OLED 显示漏水、网页告警、蜂鸣器短提醒 |
| 火源异常 | 火焰 `GPIO6` | 按实板模块触发电平标定 | 蜂鸣器短报警，OLED 与网页提示“请人工确认” |
| 离家入侵 | PIR `GPIO5` | away 模式有人经过 | 蜂鸣器短报警，OLED 与网页显示入侵告警 |

演示时不要使用明火，不要接高压负载。MQ-2 可用安全模拟方式或数值旋钮/mock board 展示。

### 4.4 语音白名单

语音不直接执行自由文本，统一转成安全意图：

| 语音示例 | 意图 | 动作 |
| --- | --- | --- |
| 开始学习 | `setMode:study` | 切到学习模式 |
| 回到居家 | `setMode:home` | 切到居家模式 |
| 我要出门 | `setMode:away` | 切到离家模式 |
| 开启节能 | `setMode:energy` | 切到节能模式 |
| 家里安全吗 | `querySafety` | 回读 MQ-2、漏水、火焰、PIR 状态 |
| 房间热不热 | `queryComfort` | 回读温湿度与舒适提醒状态 |
| 安静一下 | `muteReminder` | 临时关闭普通提醒蜂鸣，不关闭安全报警规则 |

语音识别失败时，网页保留文本测试和快捷按钮，仍然发布同一条 `voiceIntent`。

## 5. MQTT 与可视化设计

### 5.1 推荐连接路线

沿用初中项目中更稳的路线：

```text
N16R8 USB
  -> HTTPS Dashboard Web Serial
  -> WSS Cloud Relay
  -> MQTT Broker
  -> 其他电脑/手机网页看板
```

这样换电脑时只需要插 USB、打开 HTTPS 网页、点击“连接串口”。限制也要写清楚：插着 N16R8 的电脑浏览器页面必须保持打开；Safari/iPhone 可以看远程页面，但通常不能作为 USB 直连网关。

备用路线：

```text
N16R8 USB
  -> 本地 Python 网关
  -> MQTT Broker / WSS Relay
  -> Dashboard
```

后续无线化路线：

```text
N16R8 Wi-Fi
  -> MQTT Broker
  -> Dashboard
```

无线 MQTT 适合作为第二阶段，不建议第一版就依赖比赛现场 Wi-Fi。

### 5.2 MQTT Topic

为避免和初中项目或另一个小学作品混用，建议本作品使用独立前缀：

```text
smartlife/primary/hk2/n16r8/hello
smartlife/primary/hk2/n16r8/telemetry
smartlife/primary/hk2/n16r8/event
smartlife/primary/hk2/n16r8/health
smartlife/primary/hk2/n16r8/command
smartlife/primary/hk2/n16r8/config
smartlife/primary/hk2/n16r8/voiceIntent
```

如果复用初中 `tools/n16r8_gateway.py` 与 `n16r8_cloud_relay.py`，需要把 topic 前缀做成配置项，从 `smartlife/junior/n16r8` 改为 `smartlife/primary/hk2/n16r8`。

### 5.3 串口 JSON 协议

固件使用 `115200` 串口，每行一个 JSON。不要把调试中文、启动说明和 JSON 混在同一行。

`hello` 示例：

```json
{"type":"hello","project":"smartlife-primary-hk2","board":"n16r8_esp32s3","profileId":"smartlife-primary-safe-energy-home-v1","deviceName":"N16R8 智慧安心节能小屋","baud":115200,"capabilities":["webSerial","mqttBridge","dashboard","voiceIntent","energyScore"],"pins":{"light":1,"mq2":2,"sound":4,"pir":5,"flame":6,"water":8,"buzzer":13,"dht":14,"oledSda":41,"oledScl":42,"relayLamp":48}}
```

`telemetry` 示例：

```json
{"type":"telemetry","mode":"study","sensors":{"light":32,"sound":18,"temperature":28.4,"humidity":56,"pir":true,"mq2":21,"water":false,"flame":false},"actuators":{"lamp":true,"buzzer":false},"alerts":[],"energy":{"score":88,"reason":"光线不足，已开启学习灯"},"health":{"profileId":"smartlife-primary-safe-energy-home-v1","mqtt":"bridge","voice":"ready","thresholdFocus":"lightThreshold"}}
```

命令示例：

```json
{"type":"command","mode":"study"}
{"type":"command","mode":"energy"}
{"type":"command","set":{"lightThreshold":35}}
{"type":"command","set":{"temperatureThreshold":29}}
{"type":"command","set":{"buzzerEnabled":false}}
{"type":"command","actuator":{"lamp":false}}
{"type":"voiceIntent","intent":"querySafety"}
```

固件收到命令必须返回 `ack`：

```json
{"type":"ack","ok":true,"message":"mode=energy"}
```

### 5.4 网页可视化首屏

Dashboard 第一屏直接做操作台，不做介绍页：

- 顶部状态：开发板、USB、WSS、MQTT、语音、最后一帧时间。
- 模式区：居家、学习、离家、节能。
- 传感器区：光照、温湿度、声音、人体、MQ-2、水滴、火焰。
- 执行器区：学习灯、蜂鸣器；OLED 与网页显示区同步反馈。
- 小屋热区：学习区、玄关、厨卫区、中控区。
- 语音中心：麦克风输入、文本测试、最近识别、执行结果。
- MQTT/事件日志：只显示真实 `hello/telemetry/ack/event`。
- 阈值面板：光照阈值、温度阈值、声音阈值、安全阈值、8 键 AD 当前焦点。

关键要求：

- 没收到真实 `hello` 或新鲜 `telemetry` 时，页面必须显示离线或等待数据，不用默认假数据冒充在线。
- 网页按钮、语音意图、8 键 AD 和 MQTT command 应最终进入同一套命令协议。
- 插板电脑负责 Web Serial 写 USB，其他设备通过 WSS/MQTT 看状态和发送安全白名单命令。

## 6. 物理模型设计

推荐搭建 50cm x 40cm 的“一间小屋四区”模型。

```text
┌──────────────────────────────┐
│ 学习区                       │
│ 光敏 + 声音 + DHT + 学习灯    │
├──────────────┬───────────────┤
│ 厨卫安全区    │ 玄关安防区      │
│ MQ-2 + 水滴   │ PIR + 蜂鸣器      │
│ 火焰模拟      │                │
├──────────────┴───────────────┤
│ 中控区：N16R8 + OLED + 8键AD/按钮 │
└──────────────────────────────┘
```

安装原则：

- 主控和线束尽量放在中控区或底板背面，评委正面看到的是生活场景和模块标签。
- N16R8 的 USB-C 朝外，方便现场 Web Serial 授权。
- OLED 面向评委，显示当前模式、核心传感器值和告警。
- 网页看板放在模型旁边，评委能同时看到实体动作和页面变化。
- 安全模块只做低风险模拟，不使用明火，不接高压。

## 8. 实施顺序

1. 冻结本文件中的任务映射、GPIO 合同、topic 前缀和协议字段。
2. 固件最小版：实现 `hello/telemetry/ack`、光照/DHT/PIR/学习灯/节能模式。
3. 完成正式安全模块：MQ-2、水滴、火焰、PIR、蜂鸣器和 OLED/网页告警状态。
4. Web Serial Dashboard：复用初中可视化框架，换成小学 profile 与 `smartlife/primary/hk2/n16r8` topic。
5. MQTT/WSS：复用初中 Cloud Relay 和 gateway，支持 topic 前缀配置。
6. 语音入口：先做网页文本/快捷按钮，再接麦克风或小智语音节点。
7. 8 键 AD：先运行单模块例程标定 `GPIO3/GPIO10` 与 ADC 区间，再接入模式和阈值。
8. 实体模型：按四区布局安装模块，隐藏线束，贴清楚标签。
9. 验收：先 mock board，再实板，最后公网 HTTPS/WSS/MQTT。

## 9. 验收清单

功能验收：

- N16R8 启动后输出 `hello`，包含 `project=smartlife-primary-hk2` 与 `profileId=smartlife-primary-safe-energy-home-v1`。
- 遥测来自真实传感器或 mock board，不用静态假数据冒充在线。
- OLED 与网页能同步显示光照、温湿度、PIR、模式、告警。
- 学习模式下，光照低自动开灯，温度或声音超阈值时发出舒适提醒。
- 节能模式下，仅在有人且光线暗时允许开灯，并显示节能原因。
- 安全异常触发时，蜂鸣器、OLED 和网页告警按优先级响应。
- 网页按钮、语音意图、8 键 AD 或本地按钮能进入同一套命令协议。
- MQTT topic 能看到 `hello/telemetry/event/health/ack`。
- 断开 USB 后，网页显示离线或重连中。

现场验收：

- Chrome/Edge HTTPS 页面可授权 CH340/USB 串口。
- 插板电脑网页能读写 USB；另一台设备能通过 WSS/MQTT 看同步状态。
- Safari/iPhone 如使用，只作为观看端，不作为 USB 直连网关。
- 语音失败时，文本测试和快捷按钮仍可演示同一条意图链。
- 继电器只控制低压灯或指示灯。
- 学习灯与蜂鸣器上电默认关闭，不误亮、不误响。
- 8 键 AD 的实际 GPIO 与 ADC 区间已经写入最终开发文档和代码。

## 10. 主要风险与处理

| 风险 | 处理 |
| --- | --- |
| 小学作品被做得过复杂 | 讲解只围绕四项小学任务，安全/MQTT/可视化作为增强展示。 |
| 现场 Wi-Fi 不稳定 | 第一阶段使用 HTTPS Web Serial + WSS/MQTT，不依赖板端 Wi-Fi。 |
| Web Serial 浏览器不支持 | 使用 Chrome/Edge；Safari/Firefox 只看远程状态。 |
| 串口被占用 | 关闭 PlatformIO Monitor、Arduino 串口监视器、本地 Python 网关后再连接。 |
| MQTT 公共 broker 不稳定 | 正式演示使用初中服务器或自有 Mosquitto，避免长期暴露无密码 1883。 |
| 8 键 AD 引脚口径冲突 | 先跑单模块例程确认 `GPIO3` 或 `GPIO10`，再冻结代码。 |
| MQ-2 伤板 | `AO` 分压/限压到 `0~3.3V`，不用 5V 模拟输出直进 ESP32 ADC。 |
| 安全演示有危险 | 火焰不用明火，继电器不接 220V，水滴下方垫防水膜。 |

## 11. 最终推荐结论

推荐本文件夹作品采用：

```text
作品名称：N16R8 智慧安心节能小屋
小学任务：数据采集 + 智能控制 + 语音交互 + 节能响应
真实硬件：N16R8 ESP32-S3 + 光/温湿/声音/PIR/MQ-2/水滴/火焰 + 学习灯/蜂鸣器/OLED
沿用初中：Web Serial + WSS Relay + MQTT + 可视化 Dashboard
```

这样既符合小学组智慧生活的任务难度，又保留了初中项目中已经打磨过的 MQTT 与可视化亮点。评委看到的不是静态网页，而是一个真实小屋模型中的传感器、执行器、语音指令和远程看板同步工作。
