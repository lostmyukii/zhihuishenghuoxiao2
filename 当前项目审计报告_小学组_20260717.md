## 一、审计结论

本项目当前统一名称为 **N16R8 智慧安心节能小屋**，项目标识为`smartlife-primary-hk2`，画像为`smartlife-primary-safe-energy-home-v1`，固件版本为`0.2.2`。

它与“香港小学作品1”的区别很清楚：作品1以学习环境为主，作品2以**家庭安心、安全提醒和按需节能**为主。模型分为学习区、厨卫安全区、玄关安防区和中控区，正式硬件包括：

- 光敏、声音、DHT温湿度、PIR人体活动；
- MQ-2、水滴、火焰模拟；
- D12低压学习灯、D13蜂鸣器和OLED；
- N16R8、Web Serial Dashboard及WSS/MQTT展示链。

项目明确不使用8键AD、风扇、舵机和RGB灯环，答辩中不能把这些模块讲回来。

综合当前代码、自动测试、7月12日实板记录和今天的部署复核，结论是：

> **真实数据读取和串口协议已有真板证据；智能控制、语音交互、节能响应的软件逻辑完整，但比赛主稿中的实体灯、蜂鸣器、安全触发和真人语音动作，仍必须在当前成型作品上再做一次现场终验。**

所以本项目不是“没有做出来”，而是已经进入**比赛演示可靠性收口阶段**。此时最重要的不是继续加模块，而是把四条小学任务逐条做成可重复、可看见、能解释的闭环。

## 二、采用的小学组要求口径

全国青少年信息素养大赛官网“实践任务”栏目列出了2026-04-20发布的“场景：智慧生活（规则视频）”。官网入口：<https://ceic.kpcb.org.cn/cms/sxsm/index.htm>。

本文件夹两份视频分析材料将小学组能力归纳为：

1. **数据采集**：采得到、读得稳、表达清楚；
2. **智能控制**：根据光照或温度等阈值自动响应，不用按键冒充自动；
3. **语音交互**：语音指令准确触发功能，并反馈执行结果或当前状态；
4. **节能响应**：在合适时机启停设备，并能解释为什么节能。

规则视频同时说明演示仅作参考、具体任务以现场公布为准。因此本项目要训练的是“采什么、怎么判、怎么动、怎么显示、怎么解释”，而不是死背固定动作。

## 四、项目最有竞争力的内容

### 1. 一个生活故事贯穿全部功能

```text
学习区看舒适和照明
厨卫区看烟雾、漏水和火焰风险
玄关看离家时的异常活动
中控区负责判断、显示、提醒和通信
```

这比“摆了很多传感器”更有说服力。答辩时应先讲家中的真实问题，再讲硬件如何解决，不要先背GPIO编号。

### 2. 安全优先级明确

MQ-2超阈值、水滴、火焰和离家PIR能够覆盖普通控制。`actuator.buzzer=false`只停止手动测试蜂鸣，只有显式设置`buzzerEnabled=false`才关闭安全声音。这个设计能证明孩子理解了“安全功能不能被普通按钮随便覆盖”。

### 3. 真板在线与网页在线分开

Dashboard只有收到新鲜`hello/telemetry`才显示开发板在线。WSS或MQTT在线不能冒充真板在线；断开USB后应恢复等待状态。这是展示系统真实性的重要证据。

### 4. 节能原因可解释

```text
无活动 -> 灯关闭
光照充足 -> 灯关闭
有活动且光线暗 -> 灯开启
```

网页的“节能分”是程序规则评价，不是电表测得的瓦时或千瓦时。答辩重点应放在传感器、开关动作和原因三者一致。

## 五、当前主要扣分风险

### P0：正式答辩前必须解决

1. **实体执行器未形成当前终验记录**：需要确认D12继电器实际高低电平、低压学习灯、D13蜂鸣器和OLED画面均与遥测一致。
2. **真人语音闭环未留档**：必须完成“孩子说话 -> 页面出现识别文字 -> 白名单意图 -> 真板`ack` -> 模式或实体动作”。
3. **安全触发未完整实测**：水滴和火焰目前只确认正常态；必须使用湿棉签、数字信号模拟等安全方式分别触发，不使用明火和危险气体。
4. **当前没有真板在线**：今天电脑未枚举到`/dev/cu.usbserial-*`，因此今天只复核了代码、测试和公网文件，没有重新通电证明当前成型作品。

### P1：影响高分表现

1. 浏览器已有语音识别和视觉`ack`，但代码中没有语音合成播报；当前只能说“页面反馈结果”，不能说“系统已经语音回答”。
2. 光照自动控制只有单阈值，没有上下阈值或延迟防抖。正式演示要把阈值与现场明暗拉开，避免临界点反复开关。
3. 公网WSS/MQTT服务器链通过，但真实USB到第二台设备的双设备终验仍未完成。五分钟内不必把它作为主任务，除非赛前复验稳定。
4. MQ-2需要上电预热60秒。必须在安全演示前完成预热；若现场从断电开始演示，应先通电，把安全段安排在60秒以后，不能把`warming`状态误讲成检测已就绪。

- “声音百分比就是标准分贝”，或“MQ-2百分比就是专业燃气浓度”。
